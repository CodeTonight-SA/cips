# Cursor IDE Documentation Resources Guide

## Overview

This guide provides a comprehensive list of the best custom documentation resources and links to add to Cursor IDE's "@Docs" feature. These resources enhance AI-assisted coding by providing comprehensive context for web development workflows.

**Last Updated**: 2025-01-14  
**Source**: Research from Cursor's official documentation, developer communities, and best practices

---

## How Cursor Docs Work

From Cursor's official documentation:

- Type `@Docs` in chat or Composer to access documentation
- Cursor includes popular documentation by default
- You can add custom documentation by pasting URLs
- Cursor automatically crawls and indexes all subpages
- Documentation can be shared with teams
- Manage all docs in **Settings > Indexing & Docs**

## Step-by-Step Implementation Guide

### Step 1: Add Core Documentation (Priority 1-2)

1. Open Cursor IDE
2. Type `@Docs` in chat or Composer
3. Click **"Add new doc"** button
4. Paste the following URLs one by one:
   - `https://react.dev`
   - `https://nextjs.org/docs`
   - `https://www.typescriptlang.org/docs`
   - `https://tailwindcss.com/docs`
   - `https://ui.shadcn.com`
   - `https://www.radix-ui.com/primitives/docs`
   - `https://lucide.dev`

5. Wait for Cursor to index each documentation site (may take a few minutes)

### Step 2: Add Development Tools (Priority 3-4)

1. Continue adding documentation:
   - `https://react-hook-form.com`
   - `https://zod.dev`
   - `https://date-fns.org/docs`
   - `https://vitest.dev`
   - `https://playwright.dev/docs`

## Best Practices

### 1. Start Small

Begin with Priority 1-2 documentation, then expand based on your needs.

### 2. Regular Updates

Documentation sites update frequently. Periodically refresh or re-add documentation to get the latest content.

### 3. Team Sharing

Enable **"Share with team"** for documentation that your entire team uses. This ensures consistency across the team.

### 4. Project-Specific Focus

Add documentation relevant to your specific tech stack. Don't add everything - focus on what you actually use.

### 5. Remove Unused Docs

Periodically review and remove documentation you no longer use to avoid context bloat.

### 6. Test After Adding

Always test that the documentation is accessible and searchable after adding it.

## Troubleshooting

### Issue: URL Not Working

### Solution

- Ensure the URL is publicly accessible
- Check that it doesn't require authentication
- Verify the URL format is correct (include `https://`)

### Issue: Docs Not Indexing

### Solution

- Wait 5-10 minutes for Cursor to crawl and index
- Check Settings > Indexing & Docs for status
- Try removing and re-adding the documentation

### Issue: Too Many Docs

### Solution

- Focus on documentation you actively use
- Remove documentation for technologies you don't use
- Start with Priority 1-2, then add others as needed

### Issue: Documentation Not Found in Search

### Solution

- Ensure the documentation was successfully added
- Try using more specific search terms
- Check that the documentation site has proper structure for crawling
